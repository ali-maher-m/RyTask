Every capability gets one of three badges: <StatusBadge status="available" /> means it is in
the code today, <StatusBadge status="in-progress" /> means it is being built right now, and
<StatusBadge status="coming-soon" /> means it is planned but not started. This page is
maintained against the code, not the marketing copy — the tier column shows where each
capability sits on the roadmap (MVP, v2, v3).

## The nine differentiators

| ID | Differentiator | In one sentence |
|---|---|---|
| D1 | Non-technical-friendly UX | The whole product must pass the "Albert/Marissa test" — usable by a non-technical teammate without training. |
| D2 | First-class Slack capture | Turning a Slack message into a tracked task takes seconds, not a context switch. |
| D3 | MCP with 100% workspace control | Anything a human can do in the UI, an AI agent can do through the MCP server. |
| D4 | GitHub integration | Branch, PR, and commit linking with auto-close and status sync, free from day one. |
| D5 | Real planning primitives | Start, end, and due dates plus estimates, building toward a true dependency-aware timeline. |
| D6 | Native time tracking + honest reporting | A one-click timer, planned-vs-interruption tagging, and reports that show where the week actually went. |
| D7 | Priorities, custom workflows, multiple views | Custom statuses with categories and Board/List views with non-technical-friendly defaults. |
| D8 | One-command self-host | `docker compose up` and you are live, with safe transactional migrations. |
| D9 | Engineering depth | Automations, custom fields, cycles, milestones, sub-tasks, and dependencies — without metering. |
