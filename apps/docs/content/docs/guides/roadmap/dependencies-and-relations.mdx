<ComingSoon tier="MVP scope" />

RyTask plans explicit **dependencies** between work items: mark one item as blocking
another and both sides show the link. Dependencies are paywalled in Asana and Monday and
absent in Notion — here they belong to the core feature set. Softer, non-blocking links
and schedule-aware behavior follow later.

What's planned:

- **Blocks / blocked-by links** — explicit blocking dependencies between items (MVP scope).
- **Relates-to and duplicate links** — non-blocking relations between items (v2).
- **Dependency-aware scheduling** — dates shift when a blocker moves; almost no modern tool does this well (v3).
