<ComingSoon tier="v2" />

RyTask's AI position is deliberate: instead of bolting AI onto the product, it ships a
**full-control MCP server** — anything a person can do in the UI, an AI agent can do
through MCP. The agent brings the intelligence; RyTask exposes the control. That part is
not roadmap — it works today, with 49 tools covering the workspace. See the
[MCP guide](/docs/guides/mcp).

Native in-app AI features are intentionally later and intentionally modest. The lesson
from tools that led with AI autonomy is that trust has to form first — so summarize and
draft come before triage, and triage before anything autonomous.

What's planned natively:

- **AI summarize / draft** — generate descriptions and summaries (v2, via MCP first).
- **AI triage / dedup** — auto-categorize incoming items (v3).
- **AI standup / status rollups** — auto-summarize progress (v3).
- **Natural-language automations** — describe a rule in English (v3).
